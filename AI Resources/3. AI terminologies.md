## [18:11](https://www.youtube.com/watch?v=OYvlznJ4IZQ&t=1091s) 9. Retrieval Augmented Generation

**RAG** stands for **Retrieval-Augmented Generation**.  
It is a technique used in AI systems (especially with **LLMs like GPT, Claude, etc.**) to make answers **more accurate by retrieving real data before generating a response**.

Think of it as **“Search + AI writing together.”**

#### Simple Explanation (Like you're 5)

Imagine you ask a smart robot a question.

Instead of answering from memory, the robot:

1. **Searches a library** for relevant books.
2. **Reads the useful pages.**
3. **Writes an answer using that information.**

That process is **RAG**.
#### Why RAG is Needed

Normal LLMs have problems:

- They **don't know your company data**
- They **hallucinate (make up answers)**
- Knowledge is **limited to training data**

RAG solves this by allowing the AI to **read your data before answering**.
#### RAG Architecture (Step by Step)

#### 1️⃣ User Question

User asks something.

Example:

> "What is our company leave policy?"

---

#### 2️⃣ Convert Question → Embedding

The question is converted into a **vector (numerical representation)**.

Example tools:

- OpenAI embeddings
- Sentence transformers
- BERT embeddings

---

#### 3️⃣ Search Vector Database

The system searches a **vector database** for similar content.

Common vector DBs:
- Pinecone
- Weaviate
- FAISS
- Elasticsearch
- ChromaDB

---

#### 4️⃣ Retrieve Relevant Documents

The most relevant documents are retrieved.

Example retrieved text:

Leave policy:  
Employees get 20 paid leaves per year.  
Carry forward allowed up to 5 days.

#### 5️⃣ Send Context to LLM

The retrieved documents are added to the prompt.

Example prompt:

Context:  
Employees get 20 paid leaves per year.  
  
Question:  
What is our leave policy?

---

#### 6️⃣ LLM Generates Answer

Now the LLM generates an answer using the **retrieved information**.

Final answer:

> Employees receive **20 paid leaves per year**, and **up to 5 days can be carried forward**.

---

#### RAG Pipeline Diagram

```
User Question  
      │  
      ▼  
Convert to Embedding  
      │  
      ▼  
Search Vector Database  
      │  
      ▼

      │  
      ▼

      │  
      ▼  
LLM Generates Accurate Answer
```

**MCP Server (Model Context Protocol Server) — Complete Explanation**

MCP is a new architecture for connecting AI models (like LLMs) with external tools, data, and systems in a standardized way.

It was introduced by **Anthropic** to solve a big problem:

> How can AI models reliably access tools, APIs, files, and databases without writing custom integrations every time?

Think of MCP like **USB-C for AI tools**.

Instead of building a custom connector for every tool, MCP provides one standard protocol.

---

#### 1. What MCP Means

**MCP = Model Context Protocol**

It is a standard communication protocol that allows AI models to interact with external systems.

**Components:**

| Component  | Description                              |
|------------|------------------------------------------|
| MCP Client | The AI application (Claude Desktop, IDE agent, etc.) |
| MCP Server | The system exposing tools/data           |
| LLM        | The model that decides which tool to call |

**Example Flow:**

```
User → LLM → MCP Client → MCP Server → Tool/API → Response → LLM → User
```

---

#### 2. Why MCP Was Created

**Before MCP:**

Every AI tool needed custom integrations.

Example:
- ChatGPT → GitHub integration
- ChatGPT → Slack integration
- ChatGPT → Database integration
- ChatGPT → Google Drive integration

Each one needed separate code.

**With MCP:**

```
ChatGPT / Claude / AI Agent
            ↓
        MCP Protocol
            ↓
       MCP Servers
   (GitHub, DB, Files, APIs)
```

Now **any model** can connect to **any tool**.

---

#### 3. What an MCP Server Does

An MCP Server exposes **capabilities** to AI models.

**Capabilities include:**

**1️⃣ Tools** — Functions the AI can call.

Example:
```
create_invoice()
send_email()
deploy_service()
search_database()
```

**2️⃣ Resources** — Data the model can access.

Example:
- files
- documents
- knowledge bases
- APIs

**3️⃣ Prompts** — Reusable prompt templates.

Example:
- "Write a PR description"
- "Generate test cases"
- "Create architecture diagram"

---

#### 4. MCP Architecture

```
                User
                 │
                 ▼
            AI Application
         (Claude Desktop / IDE)
                 │
            MCP Client
                 │
     ┌───────────┼────────────┐
     ▼           ▼            ▼
MCP Server   MCP Server   MCP Server
(Github)       (DB)        (Slack)
     │           │            │
  Tools        Queries      APIs
```

Each MCP server exposes: **Tools**, **Resources**, **Prompts**

---

#### 5. MCP Server Example

Example MCP server exposing a database tool.

**Tool definition:**

```json
{
  "name": "search_customer",
  "description": "Search customer database",
  "input_schema": {
    "type": "object",
    "properties": {
      "email": {
        "type": "string"
      }
    }
  }
}
```

When user asks:

> Find customer with email xyz@gmail.com

The LLM will call:

```
search_customer(email="xyz@gmail.com")
```

#### 8. MCP vs Function Calling

| Feature      | Function Calling | MCP         |
|--------------|------------------|-------------|
| Scope        | Single app       | Multi-system|
| Discovery    | Static           | Dynamic     |
| Tools        | Local            | Remote      |
| Architecture | Simple           | Distributed |

> MCP is basically **Function Calling + Tool Registry + Data Access Protocol**.

---

#### 9. How MCP Server Communicates

MCP typically uses **JSON-RPC** over **STDIO** or **WebSockets**.

**Example request:**

```json
{
  "method": "tools/list"
}
```

**Response:**

```json
{
  "tools": [
    "search_customer",
    "create_invoice"
  ]
}
```

---

#### 10. Simple MCP Server Example (Node.js)

```javascript
import { Server } from "@modelcontextprotocol/sdk/server";

const server = new Server({
  name: "my-tools",
  version: "1.0"
});

server.tool(
  "getWeather",
  {
    city: "string"
  },
  async ({ city }) => {
    return `Weather in ${city} is 30°C`;
  }
);

server.start();
```

Now any MCP client can call `getWeather()`.
